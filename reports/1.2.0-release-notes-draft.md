# Beam 1.2.0 Release Notes

Status: published as `v1.2.0` on `2026-04-02`

## Summary

Beam `1.2.0` turns the current OpenClaw bridge into the first production-oriented fleet connector: one central Beam control plane can now approve hosts, ingest inventory and heartbeats, track route health, surface duplicate identity conflicts, and coordinate live cross-host delivery with Beam-managed identities.

## Highlights

- introduce first-class OpenClaw host enrollment, manual approval, revocation, heartbeat tracking, inventory sync, and route-event ingestion in the Directory admin surface
- add a dedicated OpenClaw Fleet dashboard view with host health, approval state, route totals, stale/conflict counts, and jump-offs into workspace and trace surfaces
- extend workspace identities with host metadata including host label, host health, route source, and runtime session state so operators can reason about real route ownership
- unify the host-side tooling behind `beam-openclaw-host`, keeping the OpenClaw integration external while covering receiver, inventory sync, spawn-hook sync, and local route tracking
- prove real three-host message delivery, duplicate route conflict blocking, and revoke-based remediation through repo-visible operator and fleet smoke drills
- keep Beam as the identity and policy authority: OpenClaw hosts report routes, but Beam remains the source of truth for partner permissions, outbound control, and traceability

## Operational Notes

- host enrollment is intentionally manual: a new host starts in `pending` until an operator approves it and Beam issues the host credential,
- duplicate Beam identities across hosts are treated as a routing conflict and block delivery until the operator resolves the conflict or revokes the offending host,
- revoked hosts remain visible for audit and workspace context, but their routes stop accepting delivery immediately,
- the OpenClaw integration remains an external connector/daemon; Beam does not patch OpenClaw core in this release,
- Beam still issues and manages the stable Beam IDs for imported OpenClaw identities; this release does not add a free-form custom DID editor.

## Compatibility

No protocol version bump; Beam stays on `beam/1` while extending the control plane around multi-host OpenClaw routing.
