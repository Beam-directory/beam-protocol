# Beam 1.6.0 Release Notes

Release status: `draft`

## Summary

Beam `1.6.0` is the release where the OpenClaw fleet stops being “feature-complete” and starts being operationally credible. The theme is not new protocol breadth. The theme is production readiness: external alerting, enforced roles, recovery discipline, rollback safety, and enough evidence that a three-host fleet can survive normal damage without improvisation.

## Highlights

### External fleet alerting

- let fleet exceptions leave the dashboard through real webhook delivery
- persist alert targets, tests, deliveries, and escalation history inside Beam
- make stale hosts, failed delivery paths, duplicate conflicts, and revoked-host blocks visible both in the UI and off-platform

### RBAC and destructive-action guards

- enforce `viewer`, `operator`, and `admin` across the fleet surface
- keep read-only inspection available while protecting rotate, revoke, recover, conflict-resolution, and other risky actions
- require explicit confirmation phrases for destructive paths and preserve operator audit visibility

### Recovery and rollback drills

- prove central backup and restore against the real three-host fleet harness
- recover a revoked host without rebuilding workspace identity state from scratch
- keep connector rollback and recovery state visible instead of relying on operator memory

### Soak-backed fleet confidence

- commit an accelerated seven-day fleet soak with real heartbeats, route changes, maintenance state, credential motion, and recoveries
- prove that the same fleet can survive stale-host, conflict, drain, revoke, and recovery events in one continuous harness run

### Evidence-first production cut

- commit fresh buyer, operator, incident, recovery, and soak evidence under `reports/`
- treat production-readiness as a gated release story instead of a qualitative claim

## Compatibility Note

No protocol-family change in this release train. Beam `1.6.0` remains on `beam/1`.
