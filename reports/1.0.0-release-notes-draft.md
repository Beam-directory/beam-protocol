# Beam 1.0.0 Release Notes

Status: draft (target release window: `2026-06-29`)

## Summary
Beam `1.0.0` turns Beam from a strong design-partner system into a first-production-partner system. The release is scoped around one concrete cross-company workflow, a procedural onboarding and go-live path, an operator surface that shows partner health and incidents directly, and a release discipline backed by proof packs, drills, and dry runs.

## Highlights
- formalize the first production workflow contract and go-live checklist so the product, docs, and operator motion all point at one boring external workflow.
- add partner health, SLA-risk, incident attribution, and a dedicated Partner Ops surface in the dashboard.
- add recurring operator digest and reminder delivery paths so weekly follow-through is productized instead of spreadsheet-driven.
- export a redaction-safe proof pack from live evidence for external stakeholders.
- rehearse backup/restore, environment parity, and fire-drill paths with repo-visible evidence before the final `1.0.0` cut.
- require buyer, operator, and production-partner dry runs plus explicit RC gates before the tag is allowed to ship.

## Operational Notes

- The operator digest delivery path is wired end to end and reports `delivery_unavailable` in local harness runs when SMTP or Resend is intentionally absent.
- The backup/restore drill now snapshots SQLite `db`, `-wal`, and `-shm` files so recovery evidence reflects the real WAL mode used by Directory and Message Bus.
- Proof-pack export is validated in the operator, fire-drill, and production-partner dry runs.

## Compatibility
No protocol version bump. Beam `1.0.0` remains on `beam/1`.
