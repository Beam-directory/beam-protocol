- duplicate identity conflict resolution and route-owner actions

## Gates

- [x] Three approved hosts are visible in one central Beam fleet view.
- [x] Each host can publish inventory and heartbeat successfully.
- [x] Host credential rotation returns a new credential and the host can return to `ready`.
- [x] A revoked or lost host can be recovered without rebuilding workspace identities.
- [x] Duplicate Beam identity conflicts are detected, block delivery, and are resolvable through route-owner actions.
- [x] Revoked hosts stop delivery immediately.
- [x] Linux/systemd install, status, and uninstall semantics are exercised by a repo-owned smoke.
- [x] Fleet digest surfaces stale hosts, pending credential work, duplicate conflicts, and delivery follow-up items.
- [x] Buyer/operator/fleet evidence is committed under `reports/`.
- [x] Monorepo build, tests, E2E, and docs build are green on the candidate.
