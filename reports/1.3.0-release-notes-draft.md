# Beam 1.3.0 Release Notes

Release status: `published`

## Summary

Beam `1.3.0` turns the OpenClaw fleet connector from a strong first release into a day-2 operator surface. The headline is not a new protocol layer; it is safer, more explicit operations for a multi-host OpenClaw fleet under one central Beam control plane.

## Highlights

### Host credential rotation and recovery

- rotate host credentials without rebuilding workspace identities
- recover or replace a revoked or lost host with a new credential cutover
- keep host health, credential age, and recovery state visible in the Fleet dashboard

### Duplicate identity conflict resolution

- detect duplicate Beam identities across hosts and block delivery while the conflict is unresolved
- prefer a canonical route owner, disable shadow routes, or reset owner resolution from the operator surface
- retain auditability for operator decisions around route ownership

### Linux/systemd parity

- `beam-openclaw-host` now behaves as a first-class Linux systemd user service
- Linux install, status, and uninstall semantics are exercised by a repo-owned smoke
- the default Linux secret path stays file-backed with `0600` semantics for this release

### Fleet digest and escalation loop

- add a fleet digest surface for stale hosts, pending credential work, duplicate conflicts, and delivery follow-up items
- add a repo-owned fleet digest script so the operator loop is reproducible and not memory-based

### Evidence-driven release

- repo-visible buyer, operator, fleet, digest, and Linux parity evidence lives under `reports/`
- release gating is tied to explicit proof instead of ad hoc confidence

## Compatibility Note

No protocol-family change in this release train. Beam `1.3.0` remains on `beam/1`.
