# Beam 1.7.0 Release Notes

Release status: `draft`

## Summary

Beam `1.7.0` is the hosted-fleet-adoption release. The core protocol and fleet control plane are already strong; this release is about turning that capability into something an operator can actually install, enroll, inspect, and hand off without living inside the repo.

The theme is adoption, not more protocol surface.

## Highlights

### Packaged host installer

- package the OpenClaw host bootstrap path into a copy-paste install flow
- keep local developer shortcuts, but make the primary host story explicit and reusable
- carry bootstrap tokens and enrollment context all the way into the onboarding path

### Guided enrollment and operator onboarding

- expose guided enrollment links directly in the fleet UI
- show recent enrollment requests and install-pack commands to the operator
- turn host approval into a visible product path instead of CLI-only knowledge

### Operator/member administration

- surface `viewer`, `operator`, and `admin` management in Settings
- keep destructive or sensitive fleet actions guarded while still letting viewers inspect the system
- preserve sign-in-link and shared-admin posture inside the product

### Visual regression discipline

- add screenshot baselines for the key dashboard surfaces
- run compare/report in CI as a repo-owned gate
- keep the operator UI from silently regressing while the product becomes more polished

### Supportability and analytics

- add fleet analytics summaries for host posture and route churn
- add support-bundle export so an operator can package host/workspace/trace context into one artifact
- make hosted adoption easier to debug and easier to hand off

## Release Gate

`1.7.0` is still blocked on one explicit external gate:

- external dogfood and adoption evidence (`#196`)

Internal candidate evidence is already committed, but the release should not be tagged until the hosted-adoption path is proven outside the local maintainer loop.

## Compatibility Note

No protocol-family change in this release train. Beam `1.7.0` remains on `beam/1`.
