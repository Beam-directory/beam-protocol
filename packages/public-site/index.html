One workflow on purpose

Beam gets sharper when the story stays narrow.

The current wedge is not "all agents everywhere." It is one company handing work to another company's agent, with enough trust and observability to survive real operations.

Acme starts the handoff

procurement@acme.beam.directory sends quote.request with a signed payload, timestamp, and nonce.

Beam validates and routes

The directory resolves identities, checks ACLs, enforces replay protection, and records lifecycle stages for operators.

Northwind completes the work

The partner desk fans into inventory.check, assembles the quote, and sends a signed result back to Acme.

Finance gets an async preflight

The message bus accepts purchase.preflight for durable follow-up, with retries and dead-letter visibility when the path degrades.

What ships today

Beam is already more than a protocol document.

The core repo now has enough substance that the next job is product surface: make the hosted demo, proof reports, and operator workflow impossible to miss.

Directory and trust

Identity, ACLs, nonce lifecycle, admin sessions, audit, shield policy, and observability APIs.

  • Beam IDs and DID resolution
  • Signed HTTP and WebSocket handoffs
  • Public endpoint abuse controls
  • Trace and audit APIs for operators

Durable routing

Message bus semantics for retries, dedupe, restart recovery, and dead-letter handling.

  • Nonce dedupe and retry backoff
  • Delivery accepted vs terminal ack semantics
  • Restart recovery for in-flight work
  • Dead-letter visibility in the dashboard

SDKs and demo surface

TypeScript, Python, CLI, dashboard, docs, public site, and a seeded hosted demo path.

  • Archived compatibility fixtures in CI
  • Hosted quickstart and dogfood scripts
  • Operator runbook and readiness reports
  • Public docs aligned to the same funnel
