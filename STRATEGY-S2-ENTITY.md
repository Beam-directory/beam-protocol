# S2: Beam Protocol — Eigene Entity

## Problem
Beam Protocol ist aktuell ein Side-Project von COPPEN GmbH.
Kein Enterprise wird kritische Infrastruktur auf ein Projekt ohne eigene Rechtspersönlichkeit bauen.

## Optionen

### Option A: Beam Protocol UG (haftungsbeschränkt) — EMPFOHLEN
- **Gründung:** 1 Tag beim Notar, €1 Stammkapital, ~€500 Notarkosten
- **Geschäftsführer:** Tobias Kub
- **Sitz:** Bad Dürkheim oder Berlin
- **Vorteile:** Schnell, günstig, deutsch, EU-basiert, kann in GmbH umgewandelt werden
- **Nachteile:** "UG" klingt klein, internationale Partner erwarten Inc./Ltd.

### Option B: Beam Protocol GmbH
- **Gründung:** 2-3 Wochen, €25K Stammkapital, ~€1.500 Kosten
- **Vorteile:** Seriöser, bekannte Rechtsform international
- **Nachteile:** Kapitalanforderung, langsamer

### Option C: Open Source Foundation (Apache/Linux Foundation)
- **Vorteile:** Maximale Neutralität, Enterprise-Vertrauen, Community Governance
- **Nachteile:** 6-12 Monate Bewerbungsprozess, Kontrollverlust, passt nicht in Early Stage
- **Timing:** Später (nach Product-Market Fit), nicht jetzt

### Option D: Delaware C-Corp (US)
- **Vorteile:** VC-Standard, internationale Anerkennung
- **Nachteile:** US-Steuerkomplexität, Registered Agent nötig, teurer
- **Timing:** Wenn US-VCs oder US-Markt Priorität wird
