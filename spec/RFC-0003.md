# RFC-0003: Standard Intent Types

| Field | Value |
| --- | --- |
| RFC | 0003 |
| Title | Standard Intent Types |
| Status | Draft |
| Version | 0.5.0 |
| Date | 2026-03-08 |

## 1. Summary

This RFC defines a standard starter catalog of cross-agent intent types for general queries, bookings, payments, support, and notifications.

All schemas in this RFC are JSON Schema Draft 2020-12 fragments suitable for validating the `payload` or `params` object of a Beam intent.

## 2. General queries

### `query.text`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "text": { "type": "string", "minLength": 1 },
    "language": { "type": "string", "default": "en" },
    "context": { "type": "object", "additionalProperties": true }
  },
  "required": ["text"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "text": "Find the cheapest train from Berlin to Munich tomorrow.",
  "language": "en",
  "context": { "travellers": 1 }
}
```

### `query.structured`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "query": { "type": "string", "minLength": 1 },
    "filters": { "type": "object", "additionalProperties": true },
    "sort": { "type": "array", "items": { "type": "string" } },
    "limit": { "type": "integer", "minimum": 1, "maximum": 100 }
  },
  "required": ["query"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "query": "orders",
  "filters": { "status": "open", "country": "DE" },
  "sort": ["-created_at"],
  "limit": 25
}
```

## 3. Booking intents

### `booking.request`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "resource_type": { "type": "string" },
    "resource_id": { "type": "string" },
    "start_at": { "type": "string", "format": "date-time" },
    "end_at": { "type": "string", "format": "date-time" },
    "customer": { "type": "object", "additionalProperties": true },
    "notes": { "type": "string" }
  },
  "required": ["resource_type", "start_at", "end_at"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "resource_type": "hotel_room",
  "resource_id": "berlin-central-deluxe",
  "start_at": "2026-03-10T15:00:00Z",
  "end_at": "2026-03-12T11:00:00Z",
  "customer": { "name": "Alex Doe" },
  "notes": "Late check-in requested"
}
```

### `booking.confirm`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "booking_id": { "type": "string" },
    "confirmation_code": { "type": "string" },
    "price": { "type": "number", "minimum": 0 },
    "currency": { "type": "string", "minLength": 3, "maxLength": 3 }
  },
  "required": ["booking_id", "confirmation_code"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "booking_id": "bk_12345",
  "confirmation_code": "ACME-7H2K9",
  "price": 319.0,
  "currency": "EUR"
}
```

### `booking.cancel`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "booking_id": { "type": "string" },
    "reason": { "type": "string" },
    "requested_by": { "type": "string" }
  },
  "required": ["booking_id", "reason"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "booking_id": "bk_12345",
  "reason": "Customer changed travel dates",
  "requested_by": "alice@beam.directory"
}
```

## 5. Support intents

### `support.ticket`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "subject": { "type": "string", "minLength": 1 },
    "description": { "type": "string", "minLength": 1 },
    "priority": { "type": "string", "enum": ["low", "medium", "high", "urgent"] },
    "customer": { "type": "object", "additionalProperties": true }
  },
  "required": ["subject", "description"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "subject": "Payment failed during checkout",
  "description": "Customer sees a 3DS timeout after submitting the card.",
  "priority": "high",
  "customer": { "email": "alex@example.com" }
}
```

### `support.status`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "ticket_id": { "type": "string" },
    "include_history": { "type": "boolean", "default": false }
  },
  "required": ["ticket_id"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "ticket_id": "sup_90210",
  "include_history": true
}
```

### `support.resolve`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",

  "properties": {

  "additionalProperties": false
}
```

**Example**

```json

}
```

## 6. Notification intents

### `notification.send`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "channel": { "type": "string", "enum": ["email", "sms", "push", "webhook"] },
    "recipient": { "type": "string" },
    "template": { "type": "string" },
    "data": { "type": "object", "additionalProperties": true }
  },
  "required": ["channel", "recipient", "template"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "channel": "email",
  "recipient": "alex@example.com",
  "template": "booking_confirmed",
  "data": { "booking_id": "bk_12345", "city": "Berlin" }
}
```

### `notification.subscribe`

**Schema**

```json
{
  "$schema": "https://json-schema.org/draft/2020-12/schema",
  "type": "object",
  "properties": {
    "topic": { "type": "string" },
    "channel": { "type": "string", "enum": ["email", "sms", "push", "webhook"] },
    "target": { "type": "string" },
    "filters": { "type": "object", "additionalProperties": true }
  },
  "required": ["topic", "channel", "target"],
  "additionalProperties": false
}
```

**Example**

```json
{
  "topic": "support.ticket.updated",
  "channel": "webhook",
  "target": "https://hooks.example/support",
  "filters": { "priority": ["high", "urgent"] }
}
```

## 7. Interoperability notes

- Producers SHOULD include only the documented fields unless the receiving agent explicitly allows extensions.
- Consumers SHOULD reject invalid payloads with a structured Beam error result.
- Agents MAY add domain-specific response schemas on top of these request standards.
